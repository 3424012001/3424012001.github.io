js类

类可以很方便定义存在于实例上的成员、存在于原型上的成员、以及位于类本身的成员。
一.实例、原型和类成员
1.实例成员
每次通过new操作符调用类标识符时，都会执行类构造函数，在这个函数内部可以为新创建的实例（this）添加‘自有’属性。而且在构造函数执行完毕之后，还可以给实例继续添加新成员。每个实例都对应一个唯一的成员对象，所以所有成员都不会再原型上共享。
class Person {
constructor() {
this.name = new String(‘jack’);
this.sayName = () => console.log(this.name);
this.nicknames = [‘jake’, ‘j-dog’]
}
}
let p1 = new Person();
let p2 = new Person();

     p1.sayName();  //jack
     p2.sayName();  //jack

     console.log(p1.name === p2.name);             //false
     console.log(p1.sayName === p2.sayName);       //false
     console.log(p1.nicknames === p2.nicknames);    //false

     p1.name = p1.nicknames[0];
     p2.name = p2.nicknames[1];
     p1.sayName();        //jake
     p2.sayName();        //j-dog

     p1.color = 'red';
     console.log(p1).color;      //red

2.原型方法
为了在实例间共享方法，类定义语法把放在类块中定义的方法作为原型方法。
class Person {
constructor() {
this.locate = () => console.log(‘instance’);
}
locate() {
console.log(‘prototype’);
}
}
let p = new Person();
p.locate(); //instance
Person.prototype.locate; //prototype

在类构造函数上定义方法可行，存在于不同实例。可以在类块上定义方法，等同于在原型上定义。
注意不能在类块中给原型添加原始值或对象作为成员数据
class Person {
name: ‘jay’
}
这样会抛出错误
3.静态类方法
可以在类上定义静态类方法，这些方法通常用于执行不特定于实例的操作，也不要求存在类的实例。与原型成员类似，每个类上只能有一个静态成员。静态类成员在类定义中使用static关键字作为前缀。在静态成员中，this引用类自身。其他所有约定跟原型成员一样。
（constructor中的this指向实例； 定义在原型对象上的this指向原型；static中的this指向类本身）
class Person {
constructor() {
this.locate = () => console.log(‘instance’, this);
}
locate() {
console.log(‘prototype’, this);
}
static locate() {
console.log(‘class’, this)
}
}
let p = new Person();

     p.locate();
     Person.prototype.locate();
     Person.locate();

二.类的继承
1.继承基础
使用extends关键字，就可以继承任何拥有[[construct]]和原型的对象。这意味着不仅可以继承一个类而且还可以继承普通的构造函数（保持向后兼容）。
class Vehicle{};
class bus extends Vehicle{}; //继承类
let b = new bus();
console.log(b instanceof Vehicle); //ture
console.log(b instanceof bus); //ture

     function Person(){}
     class Engineer extends Person{};    //继承普通构造函数
     let e = new Engineer();
     console.log(e instanceof Person);    //ture
     console.log(e instanceof Engineer);  //ture
类和原型上定义的方法都会带到派生类中，this的值会反应调用相应方法的实例或类
     class Vehicle{
        //定义在类块上即定义在原型上
        idprototype(id){
            console.log(id,this);
        }
        //定义在类本身上
        static idclass(id){
            console.log(id),this;
        }
     }

     class Bus extends Vehicle{};
     let v = new Vehicle;
     let b = new Bus;

     //  this指向实例
     b.idprototype('bus');   //bus,Bus{} 
     v.idprototype('vehicle');  //Vehicle{}

     //定义在类上的成员的this指向类本身
     Bus.idclass('bus');    //bus,class Bus{}
     Vehicle.idclass('vehicle');  //vehicle,class Vehicle{}

2.super()
派生类的方法可以通过super关键字引用他们的原型。这个关键字只能在派生类中使用，而且仅限于类构造函数、实例方法和静态方法内部。
（1）在类构造函数中使用super可以调用父类的构造函数
class Vehicle {
constructor() {
this.hasEngine = true;
}
}

     class Bus extends Vehicle {
        constructor() {
            super();  //相当于super.constructor()
            console.log(this instanceof Vehicle); //ture
            console.log(this);      //Bus {hasEngine : true}
        }
     }
     new Bus();

(2)在派生类的静态方法中可以通过super调用父类上的静态方法
class Vehicle {
static identify() {
console.log(‘vehicle’);
}
}

     class Bus extends Vehicle{
        static identify(){
            super.identify();
        }
     }
     Bus.identify();  //vehicle

(3)在使用super时要注意一下几点
【1】super只能在派生类构造函数和静态方法中使用；
【2】调用super()会调用父类构造函数，并将返回的实例赋值给this；
【3】super()的行为如同调用构造函数，如果需要给父类构造函数传递参数，则需手动传入。
class Vehicle {
constructor(a) {
this.a = a;
}
}
class Bus extends Vehicle {
constructor(a) {
super(a);
}
}
console.log(new Bus(‘123456abc’)); //Bus {a : 123456abc}
【4】如果派生类没有定义类构造函数，在实例化派生类时会自动调用super()，而且会传入所有传给派生类的参数。
class Vehicle {
constructor(a) {
this.a = a;
}
}
class Bus extends Vehicle {}
console.log(new Bus(‘123456abc’)); //Bus {a : 123456abc}
【5】在类构造函数中，不能在的调用super()之前引用this，否则会报错。
【6】如果在派生类中显式定义了构造函数，则要么必须在派生类中调用super()，要么就必须在派生类中返回一个对象。
class Vehicle{}
class Car extends Vehicle {}
class Bus extends Vehicle {
constructor() {
super();
}
}
class Van extends Vehicle {
constructor() {
return {};
}
}
console.log(new Car()); //Car{}
console.log(new Bus()); //Bus{}
console.log(new Van()); // {}

自定义对象

一.工厂模式
1.含义：工厂模式就是将创建对象的语句封装在一个函数里，通过传入参数来创建特定对象，最后返回创建的对象。
function createPerson(name,age,job){ //封装了一个函数createPerson（），函数里是创建对象的语句，最终返回创建的对象
var o = new Object();
o.name = name;
o.age = age;
o.job = job;
o.sayName = function(){
alery(this.name);
} ;
return o;
}

var person1 = createPerson(‘yjf’,20,’enjineer’); //调用createPerson（）函数创建了一个 person1的对象
var person2 = createPerson(‘zy’,21,’klwfji’); //调用createPerson（）函数创建了一个 person2的对象

2.优点
可以无数次调用这个函数，而每次他都会返回包含三个属性一个方法的对象，解决了创建多个相似对象的问题。
3.缺点
没有解决对象识别问题即怎样知道一个对象的类型。

二.构造函数模式
1.构造函数模式的含义
(1)ES中的构造函数可以用来创建特定类型的对象，像Object和Array这样的原生构造函数，在运行时会自动出现在执行环境里。此外我们也可以创建自定义的构造函数，从而创建自定义对象类型的属性和方法。
function Person(name,age,job){
this.name = name;
this.age = age;
this.job = job;
this.sayName = function(){
alery(this.name);
} ;
}
var person1 = new Person(‘yjf’,20,’enjineer’);
var person2 = new Person(‘zy’,21,’klwfji’);

对比工厂模式，构造函数模式有一下几点不同：
【1】没有显示的创造对象；
【2】直接将属性和方法赋给了this对象；
【3】没有return语句；
【4】函数名Person第一个字母大写（构造函数都以一个大写字母开头，非构造函数则应该以一个小写字母开头）；
（2）构造函数本身也是函数，只不过可以用来创建对象罢了。
（3）要创建Person的新实例必须使用 new 操作符。 以这种方式调用构造函数实际上会经历以下四步：
【1】创建一个新对象；
【2】将构造函数的作用域赋给新对象；
【3】执行构造函数中的代码（即给新对象添加属性和方法）
【4】返回新对象
(4)constructor(构造函数)属性
在前面的例子中pers1和person2分别保存着Person的一个不同的实例。这两个对象都有一个constructor(构造函数)属性，该属性指向Person.
alert（person1.constructor == Person）； //true
alert（person2.constructor == Person）； //true
对象的constructor属性最初是用来标识对象类型的，但是提到对象类型还是instanceof操作符比较靠谱。在我们的例子中创建的所有对象即使Object的实例，又是Person的实例。
alert(person1 instanceof Object); //true
alert(person1 instanceof Person); //true
alert(person2 instanceof Object); //true
alert(person2 instanceof Person); //true
2.将构造函数当作函数
构造函数与其他函数的唯一区别，就在于他们的调用方式不同，任何函数，只要通过new操作符来调用，那他就可以当作构造函数；而任何函数只要不通过new操作费来调用，那他跟普通函数没什么区别。
//当构造函数使用
var person = new Person (‘yjf’,20,’enjineer’);
person.sayName（）； //‘yjf’
这是构造函数的经典用法，即使用new操作符来创建一个新对象

//当作普通函数
Person(‘yjf’,20,’enjineer’); //添加到window
window.sayName（）； //‘yjf’
在全局作用域中调用一个函数时，this对象总是指向Global对象（在浏览器中就是window对象了）。因此，在调用完函数后，可以通过window对象来调用sayName()方法，并且还返回了‘yjf’。

//在另一个对象的作用域中调用
var o = new Object（）；
Person.call(o,(‘yjf’,20,’enjineer’));
o.sayName(); //‘yjf’
也可以使用call()(或者apply())在某个特殊对象的作用域中调用Person（）函数。这里是在o的作用域中调用的，因此调用后o就有了所有属性和sayName()方法。

2.优点
创建自定义的构造函数意味着将来可以将他的实例标识为一种特定的类型，而这正是构造函数模式胜过工厂模式的地方。

3.缺点
每个方法都要在每个实例中重新创造一遍.在ECMAScript中，函数即对象，因此每定义一个函数，也就是实例化了一个对象。也就是说通过构造函数实例化的多个对象的方法，是多个不同的方法，但它们内部的代码以及实现的功能是相同的，这就造成了一定的资源浪费。
而且每个Person实例都包含一个不同的sayName实例，会导致不同的作用域链和标识符解析，但创建sayName新实例的机制还是相同的。因此每个实例上的同名函数是不相等的，以下代码可以证明：
alert(person1.sayName == person2.sayName); //false

三.原型模式
1.prototype（原型）属性
我们创建的每个函数都有一个prototype（原型）属性，这个属性是个指针，指向一个对象，而这个对象的用途是包含由特定类型的所有实例共享的属性和方法。字面意思来理解的话，那么prototype就是通过调用构造函数而创建的那个对象实例的原型对象。使用原型对象的好处是可以让所有的对象实例共享它所包含的属性和方法。换句话说不必在构造函数中定义对象实例的信息，而是可以将这些信息直接添加到原型对象中。
function Person(){
}
Person.prototype.name = ‘yjf’;
Person.prototype.age = 21;
Person.prototype.job =’student’;
Person.prototype.sayName = function(){
alert(this.name);
};

var person1 = new Person();
person1.sayName(); //‘yjf’

var person2 = new Person();
person2.sayName(); //‘yjf’

alert(person1.sayName = person2.sayName); //true
